# Campus Guard AI - 演示脚本

## 演示环境准备

### 硬件要求

- 笔记本电脑 (建议带独显)
- 网络连接
- 演示用视频文件 (1080P)

### 软件准备

```bash
# 1. 启动 Docker
docker-compose up -d

# 2. 确认服务状态
docker-compose ps

# 3. 检查健康状态
curl http://localhost:8000/health
```

## 演示流程

### 第一阶段：系统概览 (2分钟)

**操作步骤**:

1. 打开浏览器访问 http://localhost:3000
2. 展示登录页面 (如启用认证)
3. 进入主控制台

**讲解要点**:

> "这是 Campus Guard AI 校园安防系统的控制台。系统采用前后端分离架构，前端使用 Next.js + TypeScript，后端使用 FastAPI + WebSocket，流媒体核心使用 C++20 实现，支持 20 路并发视频流处理。"

**展示内容**:
- 系统总览面板
- 实时统计数字 (在线流数、今日事件、待审核告警)
- 系统健康状态指示器

---

### 第二阶段：视频流管理 (3分钟)

**操作步骤**:

1. 点击左侧菜单 "视频流"
2. 点击 "添加视频流" 按钮
3. 填写流信息:
   - 名称: "教学楼A-1F-大厅"
   - 类型: RTSP
   - URL: rtsp://... (或本地文件)
4. 点击 "保存"
5. 点击 "启动" 按钮

**讲解要点**:

> "系统支持多种视频源接入，包括 RTSP 网络摄像头、RTMP 推流和本地视频文件。我们来看一下如何添加一个新的视频流。"

> "添加完成后，系统会自动建立连接，开始解码。这里可以看到实时的 FPS、延迟和丢帧情况。"

**展示内容**:
- 视频流列表
- 添加/编辑/删除操作
- 实时状态监控 (FPS、延迟、丢帧)
- 批量操作 (启动/停止)

---

### 第三阶段：多路视频矩阵 (3分钟)

**操作步骤**:

1. 返回 "概览" 页面
2. 观察多路视频同时显示
3. 点击某个视频放大查看
4. 点击 "还原" 返回矩阵视图

**讲解要点**:

> "系统支持最多 20 路视频同时接入和处理。在概览页面，我们可以以矩阵形式查看所有视频流。"

> "当某路视频检测到异常行为时，会自动高亮提醒。点击任意视频可以放大查看详情。"

**展示内容**:
- 2x2、3x3、4x5 多种布局
- 视频实时播放
- 异常高亮提示
- 全屏查看模式

---

### 第四阶段：实时行为识别 (5分钟)

**操作步骤**:

1. 播放包含异常行为的测试视频
2. 观察检测框和跟踪 ID
3. 观察行为识别标签
4. 查看置信度分数

**讲解要点**:

> "现在我们来演示系统的核心功能：实时行为识别。系统使用 YOLO 进行人员检测，DeepSORT 进行多目标跟踪，然后使用时序模型进行行为识别。"

> "可以看到，每个人都被分配了一个跟踪 ID，系统会持续跟踪这个人的运动轨迹。当检测到异常行为时，会显示行为标签和置信度。"

> "系统支持多种异常行为识别，包括："
> - "打架斗殴（互殴）"
> - "校园霸凌（单向攻击）"
> - "跌倒/昏厥"
> - "疑似轻生/极端危险行为"
> - "吸烟/点火"
> - "异常徘徊"

**展示内容**:
- 人员检测框
- 跟踪 ID 显示
- 行为标签 (fighting, falling, smoking 等)
- 置信度分数
- 角色标注 (攻击者/受害者/旁观者)

> "告警信息包括：事件类型、严重程度、发生时间、涉及人员、置信度等。"

**展示内容**:

## 常见问题 Q&A

**Q: 系统支持多少路视频并发？**
A: 系统设计上支持最多 20 路 1080P 视频并发处理，实测 20 路时平均 FPS 保持在 20 以上。

**Q: 检测延迟是多少？**
A: 单路 1080P 视频从采集到告警的端到端延迟约为 150ms，满足实时性要求。

**Q: 支持哪些视频源？**
A: 支持 RTSP 网络摄像头、RTMP 推流和本地视频文件。

**Q: 如何保证隐私安全？**
A: 系统支持本地部署，所有数据处理在本地完成。同时支持数据脱敏和访问控制。

**Q: 模型可以更新吗？**
A: 是的，系统提供了模型二次训练接口，可以使用自己的数据进行微调。

**Q: 系统可以扩展吗？**
A: 系统采用模块化设计，各模块通过标准接口通信，可以独立扩展或替换。
