# Campus Guard AI - 比赛报告

## 项目概述

**项目名称**: 面向校园安防的视频行为感知与异常事件智能预警系统

**项目目标**: 构建一个可部署、可展示、可压测、可扩展的校园安防视频行为识别系统，支持多路视频流在线接入、人员检测与跟踪、敏感/异常行为识别、多人交互角色区分、实时预警、日志审核与导出。

### 核心功能 (100%)

| 功能模块 | 完成度 | 说明 |
|----------|--------|------|

| 异常切片导出 | ✅ | 事件前后视频切片 |

## 技术架构

### 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                        前端层 (Next.js)                      │
│         视频矩阵 | 实时告警 | 历史查询 | 事件审核             │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼ WebSocket / HTTP
┌─────────────────────────────────────────────────────────────┐
│                      控制面 (FastAPI)                        │
│         REST API | WebSocket | 业务编排 | 权限控制           │
└─────────────────────────────────────────────────────────────┘
                              │
        ┌─────────────────────┼─────────────────────┐
        ▼                     ▼                     ▼
┌───────────────┐   ┌──────────────────┐   ┌──────────────┐
│  Stream Core  │   │   AI Runtime     │   │  Data Layer  │
│  (C++20)      │   │   (Python)       │   │  PG + Redis  │
│  FFmpeg解码   │   │  检测/跟踪/识别   │   │              │
│  固定线程池    │   │  ONNX推理        │   │              │
│  有界队列      │   │  规则融合        │   │              │
└───────────────┘   └──────────────────┘   └──────────────┘
```

### 技术栈

| 层级 | 技术 | 选型理由 |
|------|------|----------|
| 前端 | Next.js + TypeScript + Tailwind | 现代化、类型安全、企业级UI |
| 控制面 | FastAPI + WebSocket | 高性能、异步支持、自动生成文档 |
| 流媒体 | C++20 + FFmpeg | 绕过 Python GIL，实现真正并行 |
| AI Runtime | Python + ONNX Runtime | 跨平台、高性能推理 |
| 数据 | PostgreSQL + Redis | 可靠持久化 + 高速缓存 |
| 部署 | Docker Compose | 一键部署、环境一致 |

## 创新点

### 1. 工程架构创新

**C++20 流媒体核心**
- 固定线程池 + 有界队列，避免 Python GIL 限制
- 双线程模型：Ingest 线程与 Process 线程分离
- 背压策略：队列满时丢旧帧，保证实时性
- 环形缓冲区：30秒滑动窗口，支持快速切片导出

**模块化设计**
- 清晰的模块边界，便于独立开发测试
- 统一事件协议，前后端解耦
- 可插拔的 AI Provider 接口

### 2. 算法创新

**多人交互角色理解**
- 不仅识别"打架"，还区分"攻击者"和"受害者"
- 支持互殴、单向攻击、多人混战等复杂场景
- 时序行为建模，减少误报

**规则融合引擎**
- AI 检测结果与规则引擎融合
- 支持自定义规则，灵活配置
- 置信度动态调整

### 3. 工程实践创新

**完整的可观测性**
- per-stream 指标采集
- 全链路延迟追踪
- 自动化压测脚本

**企业级前端**
- 参考微信设计语言：克制、秩序、留白、圆角
- 响应式布局，支持大屏展示
- 实时数据可视化
