# 前端规则

## 目标
实现一个面向校园安防场景的企业级前端控制台。

## 技术要求
- 使用 Next.js App Router + TypeScript
- 使用 Tailwind CSS + shadcn/ui
- 服务端状态与本地 UI 状态分离
- 尽量写可组合组件，不在页面中堆大量重复 JSX
- 所有页面必须有 loading / empty / error 状态
- 所有页面必须接真实 API，不做长期假数据页面

## 页面要求
- /overview：KPI、在线流、今日告警、风险趋势
- /streams：多路矩阵、状态、放大预览、筛选
- /alerts：实时告警列表、详情抽屉、处置入口
- /history：历史筛选、检索、导出
- /review：事件审核、标签修正、角色修正、日志在线修改
- /settings：流配置、阈值、ROI、区域规则配置
- /training：训练任务、模型版本、重训入口

## 多路视频展示原则
- 默认仅展示少量高质量主流
- 其他宫格使用缩略流、低帧率流或快照
- 用户点击后切换到高质量流
- 不要假设浏览器应同时全清展示 20 路满帧视频

## 交互要求
- 实时告警通过 WebSocket 更新
- 历史记录支持多条件筛选
- 表格与详情侧栏联动
- 审核页支持在线修改并写回 API
