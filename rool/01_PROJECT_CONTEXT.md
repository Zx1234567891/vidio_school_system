# 项目上下文

## 项目名称
面向校园安防的视频行为感知与异常事件智能预警

## 项目目标
构建一个可部署、可展示、可压测、可扩展的校园安防视频行为识别系统，支持多路视频流在线接入、人员检测与跟踪、敏感/异常行为识别、多人交互角色区分、实时预警、日志审核与导出、异常切片导出、历史缓存、测试接口和模型二次训练接口。

## 题目硬要求抽象
- 支持单路或多路视频在线处理
- 输入分辨率不低于 1080P
- 支持长时间连续运行
- 识别正常行为不误报
- 重点识别敏感和异常行为
- 支持单人行为与多人交互行为区分
- 支持多人交互行为中的角色理解
- 行为识别结果要有明确时间定位
- 系统流程必须模块化、便于部署与扩展
- 需要前端展示、本地视频导入、日志在线修改、日志导出、异常切片导出、历史记录缓存
- 需要开放测试接口和模型二次训练接口

## 我方额外工程目标
- 最多 20 路并发视频流接入
- 推流与处理加速层使用 C++20 + 固定线程池
- 前端风格大气、现代、企业级，视觉参考微信的克制、秩序、留白和圆角感
- 研发方式使用 Claude Code 分阶段落地

## 重点识别行为
至少覆盖以下类别：
1. 高风险异常行为
   - 打架斗殴（互殴）
   - 校园霸凌（单向攻击）
   - 跌倒/昏厥
   - 疑似轻生/极端危险行为
   - 破坏公共设施
2. 管理敏感行为
   - 吸烟/点火
   - 打电话/长时间使用手机
   - 遮挡/干扰摄像头
3. 可疑行为
   - 异常徘徊/长时间滞留
   - 翻越围栏/闯入限制区域

## 性能与工程目标
- 单路 1080P 延迟目标：< 300ms
- 支持至少 1 小时长视频稳定输入
- 系统应稳定，不出现明显卡顿或异常中断
- 输出事件要可查询、可审核、可导出
- 系统方案要完整、结构清晰、可复现
